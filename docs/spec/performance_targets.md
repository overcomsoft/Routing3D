# 성능 목표 정의 (Performance Targets) — Routing3D Phase 3 엔진

> Phase 2 인터페이스 동결 · [phase2_plan.md](../phase2_plan.md) Step 2.4
> 버전 1.0 · 2026-05-28 · 단위 **mm**, 기본 셀 50mm
> 본 목표는 Phase 3 C++ 엔진의 합격 기준이다. 사용자 확정 입력(2026-05-28)을 반영한다.

---

## 1. 확정 목표 (사용자 입력)

| 항목 | 목표 | 비고 |
|---|---|---|
| 최대 도메인 규모 | **~8,000m 스케일** | OpenVDB(희소 복셀) + 계층 corridor 필수 |
| 단일 배관 라우팅 | **< 1초 (인터랙티브)** | corridor 한정 fine 탐색 기준 |
| 전체 프로젝트(수백 배관) | **< 1분** | 다중 배관 조정 포함 |
| 메모리 상한 | **< 32GB** | 점유맵 + 탐색 자료구조 합 |

> 전체 <1분 / 수백 배관 → **평균 0.1~0.3초/배관** 필요(병렬화 활용). 단일 배관 <1초는 worst-case(혼잡/장거리) 기준.

---

## 2. 규모의 의미 (왜 Dense 가 불가능한가)

50mm 셀에서 축당 셀 수 = 길이/50mm.

| 물리 규모 | 축당 셀 | 전체(Dense) 셀 | Dense 메모리(1B/셀) |
|---|---|---|---|
| 50m | 1,000 | 10⁹ | 1 GB |
| 450m (project ROI) | 9,000 | ~7×10¹¹ | 불가 |
| 8,000m | 160,000 | ~4×10¹⁵ | **불가(페타바이트)** |

→ 8,000m 스케일은 **Dense/BitPacked 전면 그리드화 불가**. 반드시 **OpenVDB 희소 복셀**(점유/활성 영역만 저장) + **계층 corridor**(coarse→fine)로 탐색 공간을 제한해야 한다. 이것이 §1 목표 달성의 전제다.

---

## 3. 규모별 목표치 (셀/활성복셀 기준)

"활성 복셀" = OpenVDB 가 실제 저장하는 점유/관심 영역 셀. 탐색은 corridor 로 제한되어 **전체 도메인 크기와 무관하게** 한 배관의 fine 탐색량이 bounded.

| 활성 복셀 규모 | 대표 케이스 | 단일 배관 탐색 | 점유맵 메모리 |
|---|---|---|---|
| ~10⁶ | 작은 ROI | < 50 ms | < 100 MB |
| ~10⁷ | project6 전체(50mm) | < 0.3 s | < 1 GB |
| ~10⁸ | 중규모(~1,000m corridor) | < 1 s | < 8 GB |
| ~10⁹ 이상 | 8,000m(OpenVDB 희소) | < 1 s (corridor 한정) | < 32 GB |

- **단일 배관 < 1초 불변**: 계층 corridor 가 fine 탐색을 ~10⁵~10⁶ 확장으로 제한 → C++(평면배열 A* + Boost.Heap)에서 달성.
- **전체 프로젝트 < 1분**: 수백 배관을 병렬/순차 + rip-up 조정 포함. 평균 0.1~0.3초/배관.

**C++ 목표 처리량**: 평면배열 점유 + 정수 인덱스 + Boost.Heap → **수백만~천만 확장/초**(50~200×). 추가로 **계층 corridor 가 확장 수 자체를 10~100× 감축**. 두 효과로 8,000m 규모에서도 단일 배관 <1초가 현실적.

## 5. 8,000m 대응 전략 (수용 기준)

1. **OpenVDB 희소 점유맵**: 점유/활성 영역만 저장. far-field 는 coarse 해상도. 메모리 <32GB 검증을 조기 PoC 로.
2. **계층 corridor 라우팅**: coarse 그리드(예 200~500mm)에서 대략 경로 → 그 주변 corridor 만 50mm fine 탐색. fine 탐색량 bounded → 단일 배관 <1초.
3. **자료구조**: `g/closed` → 선형 인덱스 배열/`abseil flat_hash_map`, 큐 → Boost.Heap(decrease-key).
4. **병렬화**: 독립 배관 병렬 라우팅(전체 <1분), 충돌은 rip-up/CBS 단계에서 조정.
5. **충돌검사**: AABB 복셀 기본, 필요 시 FCL 메시.
